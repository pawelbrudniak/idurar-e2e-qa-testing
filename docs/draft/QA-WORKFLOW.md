# 🧪 QA Workflow Simulation -- GitHub Projects

## 📌 Overview

This project demonstrates a **real-world QA workflow simulation** built
entirely using **GitHub Projects**.

Instead of relying on commercial tools like Jira or TestRail, I designed
and implemented a **fully functional QA management system** that
replicates their core capabilities.

👉 The workflow is **live and fully interactive**, not just static
screenshots.

------------------------------------------------------------------------

## 🎯 Goal

The goal of this project is to:

-   simulate real QA processes used in professional environments
-   demonstrate practical understanding of testing workflows
-   showcase ability to design and manage QA processes independently

------------------------------------------------------------------------

## 🧰 Tools & Equivalents

This project mirrors functionality of industry tools:

  Tool                    Equivalent in this project
  ----------------------- --------------------------------
  Jira                    GitHub Issues + Projects
  TestRail                Structured Test Cases (Issues)
  Test Management Tools   Custom workflow + fields

------------------------------------------------------------------------

## 🔄 Implemented QA Workflow

The workflow is structured to reflect real QA lifecycle:

Backlog → In Progress → Testing → Done

Additionally, defect lifecycle is implemented:

New → In Progress → Fixed → Retest → Closed → Reopened

------------------------------------------------------------------------

## 🔗 Traceability Model

A full traceability chain is implemented:

Epic → Story → Test Case → Bug

This ensures:

-   clear requirement coverage
-   defect linkage to requirements
-   full visibility of testing scope
