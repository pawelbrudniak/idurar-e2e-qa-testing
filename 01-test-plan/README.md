# 📋 Test Plan

This directory contains the test strategy and scope for the iDURAR QA project.

## Scope
Testing focuses on key ERP/CRM functionalities:
- authentication
- customer management
- invoice and payment workflows

## Objectives
- verify system functionality
- identify defects
- assess usability and risks

## Approach
The testing process follows ISTQB principles:
- requirements analysis
- test design
- execution
- defect reporting

## Test Levels
- functional testing
- basic non-functional testing (usability, validation)

## Risks
- incomplete validation
- incorrect business logic
- poor error handling
